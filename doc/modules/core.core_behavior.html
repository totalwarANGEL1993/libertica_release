Goal_Deliver (_GoodType, _GoodAmount, _OtherTarget, _IgnoreCapture)
Einem Spieler müssen Rohstoffe oder Waren gesendet werden.

In der Regel wird zum Auftraggeber gesendet. Es ist aber möglich auch zu einem anderen Zielspieler schicken zu lassen. Wird ein Wagen gefangen genommen, dann muss erneut geschickt werden. Optional kann dem Spieler auch erlaubt werden, den Karren zurückzuerobern.

Parameters:

  • _GoodType string Typ der Ware
  • _GoodAmount integer Menga der Ware
  • _OtherTarget ? integer Anderes Ziel als Auftraggeber
  • _IgnoreCapture (optional) boolean Wagen kann zurückerobert werden
Goal_Diplomacy (_PlayerID, _Relation, _State)
Es muss ein bestimmter Diplomatiestatus zu einer anderen Patei erreicht werden. Der Status kann eine Verbesserung oder eine Verschlechterung zum aktuellen Status sein.

Die Relation kann entweder auf kleiner oder gleich (<=), größer oder gleich (>=), oder exakte Gleichheit (==) eingestellt werden. Exakte GLeichheit ist wegen der Gefahr eines Soft Locks mit Vorsicht zu genießen.

Parameters:

  • _PlayerID integer Partei, die Entdeckt werden muss
  • _Relation string Größer-Kleiner-Relation
  • _State string Diplomatiestatus

Goal_DestroyPlayer (_PlayerID)
Eine andere Partei muss besiegt werden.

Die Partei gilt als besiegt, wenn ein Hauptgebäude (Burg, Kirche, Lager) zerstört wurde.

Achtung: Bei Banditen ist dieses Behavior wenig sinnvoll, da sie nicht durch zerstörung ihres Hauptzeltes vernichtet werden. Hier bietet sich Goal_DestroyAllPlayerUnits an.

Parameters:

  • _PlayerID integer ID des Spielers

Goal_DestroyScriptEntity (_ScriptName)
Ein benanntes Entity muss zerstört werden.

Ein Entity gilt als zerstört, wenn es nicht mehr existiert oder während der Laufzeit des Quests seine Entity-ID oder den Besitzer verändert.

Achtung: Helden können nicht direkt zerstört werden. Bei ihnen genügt es, wenn sie sich "in die Burg zurückziehen".

    Goal_BuildWall (_PlayerID, _Position1, _Position2)
    Eine Mauer muss gebaut werden um die Bewegung eines Spielers einzuschränken. Einschränken bedeutet, dass sich der angegebene Spieler nicht von Punkt A nach Punkt B bewegen kann, weil eine Mauer im Weg ist. Die Punkte sind frei wählbar. In den meisten Fällen reicht es, Marktplätze anzugeben.

    Beispiel: Spieler 3 ist der Feind von Spieler 1, aber Bekannt mit Spieler 2. Wenn er sich nicht mehr zwischen den Marktplätzen von Spieler 1 und 2 bewegen kann, weil eine Mauer dazwischen ist, ist das Ziel erreicht.

    Achtung: Bei Monsun kann dieses Ziel fälschlicher Weise als erfüllt gewertet werden, wenn der Weg durch Wasser blockiert wird!

    Parameters:

    • _PlayerID integer PlayerID, die blockiert wird
    • _Position1 any Erste Position
    • _Position2 any Zweite Position
